# Galaxy Data Preparation Pipeline

Converts raw OASIS/ATLAS3D integral-field kinematics into modelling-ready
inputs, with optional HST image download and PSF fitting. For custom pPXF
kinematics in variant directories (e.g. `-h6`), see "New Galaxy with Custom
Kinematics" and "Replacing OASIS Kinematics" below.

## When to use

- Adding a **new galaxy** to the JAM or AxiSchw pipeline
- **Batch processing** multiple OASIS galaxies (McDermid+2006 sample)
- **Re-fitting the PSF** after downloading new HST images
- **Re-running** with different config parameters

### Required data

| Data | Location | Required? |
|------|----------|-----------|
| OASIS kinematics | `data/raw/{galaxy}/oasis/kinematics_oasis_{galaxy}.fits.gz` | Yes |
| MGE decomposition | `data/processed/sauron_mge.ecsv` OR custom MGE | Yes |
| HST F555W DRZ | `data/raw/{galaxy}/hst_*_f555w_*_pc_drz.fits` | For PSF fitting |
| SAURON C2D + IDL | `data/raw/{galaxy}/sauron/` | For `_s` files |

## Pipeline Overview

```
Step 1: Data Preprocessing
───────────────────────────
prep_oasis_batch.py ──► aperture_o / bins_o / kin_o + mge.ecsv + description.yaml
prep_sauron_batch.py ──► aperture_s / bins_s / kin_s  (sigma ×1.05)
                         └── combine _o + _s ──► gauss_hermite_kins.ecsv

Step 2: HST + PSF
─────────────────
Download HST DRZ ──► fit_psf.py (OASIS)   → psf_fit/psf_result.yaml
                     fit_psf.py (SAURON)  → psf_fit_sauron/psf_result.yaml
```

## Step 1: Data Preprocessing

### OASIS preprocessing

```python
import sys; sys.path.insert(0, 'scripts')
from prep_oasis_batch import prep_galaxy
prep_galaxy("NGC4552")
```

Output (`data/processed/{galaxy}/`):

| File | Content |
|------|---------|
| `aperture_o.dat` | Bounding box, rotation angle, grid dimensions |
| `bins_o.dat` | Flattened 2D Voronoi bin map |
| `gauss_hermite_kins_o.ecsv` | GH kinematics (v, σ, h3–h6) |
| `mge.ecsv` | MGE photometric decomposition |
| `description.yaml` | Galaxy metadata stub |

### SAURON preprocessing

```bash
cd scripts && python prep_sauron_batch.py
```

For individual galaxies, adapt `prep_sauron_batch.py`. SAURON sigma ×1.05
correction is automatically applied.

### PA Convention

| Location | Convention | Example |
|----------|-----------|---------|
| `description.yaml` `pa` | Astronomical PA (N→E, deg) | 147.0° |
| `aperture_o.dat` line 4 | `90° − PA`, CCW | −57.0° |
| SAURON `angle_deg` input | Same as aperture convention | −57.0° |

## Step 2: HST + PSF

### Download HST F555W DRZ

Only **HLA drizzled (DRZ)** products are supported. The function
`fit_psf.find_hst_file()` searches for `*f555w*_pc_drz.fits`. Non-F555W
bands (e.g. F702W) need a wrapper that patches `find_hst_file`.

```python
from astroquery.mast import Observations
obs = Observations.query_criteria(
    target_name="NGC4552", obs_collection="HST",
    filters=["F555W"], dataproduct_type="image"
)
```

**Handle non-standard target names:** galaxies with nuclear HST observations
may use `{galaxy}-NUC` as the target name. Try both:

```python
for t in [galaxy, f"{galaxy}-NUC"]:
    obs = Observations.query_criteria(target_name=t, ...)
    if len(obs) > 0: break
```

**Check multiple proposals** — some galaxies have F555W data from multiple
proposals (e.g. NGC5845: prop 2600 [no DRZ] + prop 6099 [with DRZ]).

If no HLA DRZ exists, skip PSF fitting and use FWHM ÷ 2.355 instead.

### Fit Multi-Gaussian PSF

```bash
cd JAM

# OASIS PSF
python scripts/fit_psf.py NGC4552 [--niter 50] [--ncpu 100]

# SAURON PSF (auto r<15" mask)
python scripts/fit_psf.py NGC4552 --kin-input ATLAS3D
```

Output: `results/JAM/{galaxy}/psf_fit/psf_result.yaml` containing
`sigma1`, `sigma2`, `weight1`, `chi2`.

**Non-F555W HST images:** `find_hst_file()` is hardcoded to search for
`f555w`. For other filters, monkey-patch before calling:

```python
import fit_psf
fit_psf.find_hst_file = lambda g: f"data/raw/{g}/my_hst_file.fits"
```

**Known crash:** SAURON C2D cubes often contain non-positive flux values
that cause `log10 → NaN` → `tricontour` crash in `jam_fit.prep.read_sauron_kin`.
Monkey-patch `read_sauron_kin` to sanitize flux array, or switch to use-only-OASIS
kinematics.

### Variant Directory: Custom Kinematics

When using custom pPXF kinematics in a variant directory (e.g. `-h6` suffix),
the full workflow is:

```bash
GALAXY="NGC5845"
TAG="h6"
OUT="data/processed/$GALAXY-$TAG"
mkdir -p "$OUT"
```

```python
# 1. Generate _o from pPXF FITS
create_kin_input(GALAXY, f"ppxf/output/{GALAXY}_refbins/ppxf_kins_{GALAXY}_h6.fits",
                 f"{OUT}/", expr="_o", kin_input="OASIS",
                 fit_PA=True, plot=True, min_gh_err=0.01)

# 2. Generate _s from SAURON
create_kin_input(GALAXY, [sauron_cube, sauron_kin],
                 f"{OUT}/", expr="_s", kin_input="ATLAS3D", ...)

# 3. Sigma offset
offset_datafile(f"{OUT}/gauss_hermite_kins_s.ecsv", offset={"sigma": 1.05})

# 4. Add PSF + combine
add_psf_to_datafile(sigma, weight, f"{OUT}/gauss_hermite_kins_o.ecsv")
add_psf_to_datafile(sigma, weight, f"{OUT}/gauss_hermite_kins_s.ecsv")
combine_kin_file(f"{OUT}/", ["_o", "_s"])

# 5. MGE (if available)
create_mge_table(f"{OUT}/", I, sigma_list, q_obs)

# 6. description.yaml
write_stub(GALAXY, processed_dir=OUT, overwrite=True)
```
